# ADR-0002: ブラウザ内編集に File System Access API を使う

サーバーレスのまま `wbs.json` へ書き戻すため、ブラウザ内編集の保存に File System Access API を使う。

## ステータス
承認済み（#17〜#19, #29, #60 で制約と回避を確定）

## コンテキスト
[ADR-0001](0001-dependency-free-single-file.md) によりサーバーが無い。それでも「画面から `wbs.json` を直接編集して保存したい」という要求がある。`file://` でローカルファイルに**書き戻す**手段が要る。

## 決定
編集モードの保存に **File System Access API**（`showSaveFilePicker` / `createWritable`）を使い、開いたファイルハンドルへ自動保存する。

## 理由
- サーバーレスでローカルファイルに**追記権限つきで書き戻せる**唯一の標準API。
- `file://` 前提のツールに合致（Chromium 系で動作）。

## 制約（実装で踏んだ罠 → Issueで記録）
- `file://` では権限プロンプトが出せない → 編集ON時に保存ダイアログで同じファイルを選び直す段階的フォールバック（#17 / #18 / #19）。
- ピッカー選択時点でファイルが切り詰められる → 選択直後に即保存して空ファイル化を防ぐ（#29）。
- ダイアログは参照中フォルダで開く（#60）。
- 横入り（同期/AV）の `InvalidStateError` は1回だけ自動リトライ（#74）。

## 結果
- **保存パスは「聖域」**（データ消失の実績あり）。書込は単一キューで直列化・保存前に mtime 検知・パース成功後にハンドル差替。
- Firefox / Safari は非対応（API未実装）。会社管理ブラウザは無効化で編集不可（閲覧は可）。
- この経路の変更はブランチ＋e2e＋実機スモーク必須（headless では権限挙動を再現できないため）。
