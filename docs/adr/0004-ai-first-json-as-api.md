# ADR-0004: AIを第一級ユーザーにする（JSON＝AIのAPI仕様）

人間=GUI／AI=素のJSON＋`CLAUDE.md` の2経路を第一級にし、スキーマをほぼ固定する。

## ステータス
承認済み（#67 でコンセプトを明文化）

## コンテキスト
大半のPMツールは「人間（GUI）」だけを利用者と想定する。本ツールの賭けは「AI台頭で少数精鋭がAIを使いこなすのが主流になり、リード（人間 and PL/TLエージェント）が計画を管理する」という見立て。AIが計画を読み書きする経路を一級市民にするかどうかの判断。

## 決定
**2つの第一級インターフェース**を設ける：人間＝GUI／AI＝素のJSON＋AI可読な `CLAUDE.md`。全機能がGUI経路とAI/JSON経路の**両方**で動く（カスタムキー保持・round-trip）。
`CLAUDE.md` は「ドキュメント」でなく「**AIのAPI仕様**」と位置づけ、**スキーマはほぼ動かさない**。

## 理由
- AIはフォーマットに依存して読み書きする。スキーマが揺れるとAI経路が壊れる → スキーマ固定を最優先。
- 素のJSON1ファイルなので、プラグイン・連携なしに一括編集・担当別集計・横断分析が「1文」で頼める。
- 派生値を持たせない（[ADR-0003](0003-no-derived-values-in-data.md)）ため、AIは事実（日付）だけ書けばよい。

## 結果
- スキーマ変更時は**読む側の後方互換**（旧フォーマット読み込み or 変換）を必ず用意し、`CLAUDE.md` / `CLAUDE.en.md` を同コミットで更新。
- `_` 始まりのカスタムキーはビューアが無視しつつ保存で保持（AIのメタ情報置き場：`_ai` トークン記録など）。
- 実例：v1.3 で top-level の任意フィールド `holidays` を追加した。未指定なら従来どおり土日のみ着色する後方互換を保ち、`CLAUDE.md` / `CLAUDE.en.md` を同コミットで更新した。「スキーマをほぼ固定」は変更の禁止ではなく、この手順を踏むことを条件に許す方針である。
