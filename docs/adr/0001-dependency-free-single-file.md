# ADR-0001: 依存ゼロの単一HTML・`file://` で動かす

サーバー・ビルド・外部依存を持たず、単一HTMLを `file://` で開く形態を製品の前提にする。

## ステータス
承認済み（基盤決定・v1.0 から）

## コンテキスト
WBS Viewer を配布・運用する形態の選択。サーバーを立てる／ビルドの要る SPA にする／単一HTMLにする、の3択。
対象は「AIを含む少数精鋭を率いる個人〜小規模の管理者」（[ADR-0004](0004-ai-first-json-as-api.md)）で、会社管理ブラウザ・オフライン環境も想定する。

## 決定
製品を**単一の `wbs_viewer.html`（CSS+JS内蔵）**とし、外部ライブラリ・CDN・ビルド・HTTPサーバーを使わず **`file://` でそのまま開く**。

## 理由
- **ポータビリティ**：1ファイルをダウンロードして開くだけ。サーバー運用・ビルド環境が不要。
- **サプライチェーン攻撃面ゼロ**：依存パッケージ・CDNが無いので、第三者コードの混入経路が無い。
- **会社管理ブラウザでも動く**：外部送信・追加権限が無く、閲覧はポリシーに阻まれにくい。
- **「更新＝HTML差し替え」**：データ（`wbs.json`）に触れずにツールだけ更新できる。

検討して却下：サーバー型（運用コスト・社内設置の壁）／ビルド前提のSPA（依存とビルド環境が増える）。

## 結果
- 外部送信・eval・テレメトリ・CDN・依存追加は**禁止事項**になった（依存ゼロが製品価値そのもの）。
- 編集機能はサーバーレスでローカル保存する必要があり、File System Access API を採用（[ADR-0002](0002-file-system-access-editing.md)）。
- 回帰テストは実描画を確認する headless Chromium に寄せた（[ADR-0006](0006-e2e-headless-chromium.md)）。
